1. **- docker ps -a** - список запущенных и остановленные контейнеров, без "-а" только запущенные
2. **- docker images** - список локальных образов
3. \- **docker run "hello-world" -** создается и запускается контейнер, но сперва ищет в локально репозитории образ - "hello-world", если нет, то скачивает ее
4. **- docker rm "id контейнера"** - удаляет контейнер по id
5. **- docker run -it busybox** - создается и запускается контейнер, и команда "-it" говорит запусти и открой терминал контейнера, в буквальном смысле заходит в контейнер. Внутри контейнера можно посмотреть доступ к интернету с помощью ping 8.8.8.8 или ping google.com
6. **- exit -** останавливает контейнер
7. **- docker container prune** - удаляет все контейнера
8. **- docker run -d ngnix** - создается и запускается контейнер в фоновом режиме
9. **- docker container inspect "id"** - показывает детали контейнера, там нужен был ip контейнера
10. **- docker stop "id"** - остановить контейнер запущенный в фоновом режиме
11. **- docker exec -it "id" "команда Linux"** - запуск дополнительных процесс в запущенном контейнере
12. **- docker run -d --name my\_ngnix ngnix** - запуск контейнера с кастомным именем "**my\_ngnix"**
13. **- docker run -p 8080:80 nginx** - запуск контейнера на локолхосте (докер перебрасывает с порта 8080 во внутрь контейнера)
14. **- docker run -v ${PWD}:/usr/share/nginx/html -p 8080:80 -d nginx** - запуск контейнера с маппингом томов внешних папок с папками контейнера, в данном случае html файла на рабочем столе, "**${PWD}"** это путь на внешнем устройстве и "**/usr/share/nginx/html"** это путь внутри контейнера, они маппятся через -v
15. **- docker run -it --rm busybox** - удаление контейнера после его завершение
16. **- docker run \\**
     **--name my\_container \\**
     **-d \\**
     **-rm \\**
     **nginx** - разделении команд для читабельности кода
17. - docker compose up - запуск докер компуса
18. - docker compose down - останавливает запушенный файл
19.- docker compose up -d --build - это пересобирает образы, когда внутри образа было изменение
20. - docker build -t "имя образа" . - это создание образа и присваивание ему имя, . значит откуда собирать образ
21. - docker compose:
      version: "3"
      service:
       api:
          build: ./(название папки с бэкенд)
          ports:
             '-5555:5000'
       mysql:
          image: mysql
          ports:
             '-8888:8000'
          volumes:
              -mysql_data:/var/lib/mysql (маппинг томов,где будет хранится данные)
          environment:
              MYSQL_ROOT_PASSWORD: password
              MYSQLDATABASE: time_db
       volumes: (том для хранение данных внтури докера,а не локальном комптютере)
          mysql_data
